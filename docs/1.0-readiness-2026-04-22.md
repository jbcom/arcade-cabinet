No additional imports are accepted before this set has passed the full 1.0 gate.

## Shared 1.0 Gate

Every cartridge must satisfy the same cabinet contract:

- 8-15 minute `standard` session target.
- `cozy`, `standard`, and `challenge` modes exposed before start.
- Recoverable pressure in standard mode; challenge carries the sharper stressors.
- Local save/resume for mode, progress, best score, settings, and one active run.
- Pause menu with Resume, Restart, Rules, Settings, Cabinet, and Quit Run.
- Mobile-first controls with touch-anywhere joystick or pointer/touch parity.
- Desktop and mobile Vitest Browser screenshots.
- Android debug APK and release AAB build from the same Vite output.

## Current Implementation Notes

This pass establishes the shared runtime and release shell needed to finish the per-game work consistently:

- Pure runtime models live in `src/shared/runtime.ts`.
- Browser storage and document settings live in `app/shared/hooks/useCabinetRuntime.ts`.
- Pause, settings, rules, and boot fallback UI live in `app/shared/ui/CabinetRuntime.tsx`.
- Every cartridge start screen now receives its `gameSlug`, remembers the last selected mode, and creates an active local run.
- The Android release path builds `arcade-cabinet-release.aab` and documents Play data-safety behavior.

The next per-cartridge work should close the acceptance rows above with screenshot-backed changes, one game at a time, without changing launch scope.
